Lab 01 — CLI Tour

Move around a Linux filesystem with confidence — navigate, create, inspect, copy, and clean up entirely from the shell.

Course01 — Linux Fundamentals
Time~45 minutes
Deliverable~/lab-01/transcript.txt + ~/lab-01/tree.txt + a one-paragraph reflection
Maps to slides4–9 (Part 01 — The Shell)

Goal

Sit down at any Linux box and feel oriented in under thirty seconds. You should leave this lab able to navigate, create files, view them, copy/rename, and clean up safely — without reaching for a search engine.

Setup

You need a working terminal in a real Linux environment:

  • A Linux VM (multipass, VirtualBox, UTM, Lima, etc.),
  • WSL2 on Windows,
  • A cloud VM you can SSH into, or
  • macOS — most commands work, but a few flags differ (notably ls, tail, and stat).

Open a terminal. Confirm you're in your home directory:

echo $HOME
pwd

If pwd doesn't match $HOME, run cd ~ first.

Tasks

Work through these in order. Don't paste blindly — read each line before you press Enter, especially the destructive ones.

01Orient yourself

whoami            # who am I logged in as
hostname          # what machine
pwd               # where am I
ls                # what's here
ls -la            # the whole truth — including hidden files

Now poke at the system itself — without changing anything:

ls /              # top of the filesystem
ls /etc | head    # configuration files
ls /var/log       # log files
Questions to answer for yourself: how many top-level directories does / have? Roughly how many files live in /etc?

02Build a workspace

Create a nested directory in one shot:

mkdir -p ~/lab-01/{notes,scripts,logs}
cd ~/lab-01
ls -la

You should see three directories. mkdir -p is idempotent — running it again is harmless.

03Create some files

Three ways to make a file:

touch notes/hello.md                              # empty file
echo "first line"  > notes/journal.md             # overwrite (one-line file)
echo "second line" >> notes/journal.md            # append
printf "alpha\nbeta\ngamma\n" > notes/letters.md  # multi-line

Confirm:

ls -la notes/
cat notes/journal.md
cat notes/letters.md

04View files

Try every reader on the same file:

cat notes/letters.md
head -n 2 notes/letters.md
tail -n 2 notes/letters.md
less notes/letters.md      # press q to quit; /alpha then n/N to search

Now try the live one. In terminal A:

tail -f logs/live.log

In a second terminal (open a new tab):

cd ~/lab-01
for i in 1 2 3; do
  echo "event $i" >> logs/live.log
  sleep 1
done

Watch terminal A. The new lines appear as they're written. This is how engineers watch production logs. Press Ctrl + C in terminal A when done.

05List like a pro

ls -l notes/                # long format
ls -lah notes/              # + human-readable sizes, + hidden
ls -ltr notes/              # sorted by time, oldest first
ls -lSr notes/              # sorted by size, smallest first

Read the columns of ls -l carefully — permissions, links, owner, group, size, date, name. (Slide 08 has the anatomy diagram.)

06Use the help system

Without searching the web, find:

  • The flag for cp that preserves permissions and timestamps.
  • The flag for rm that prompts before each deletion.
  • What tee does.

Use these tools:

man cp           # full manual; press q to quit
cp --help        # quick reference
whatis tee       # one-line description
type ls          # is it a binary, alias, or builtin?
which ls         # where on $PATH does it live

07Copy, rename, and move

cp notes/letters.md notes/letters-backup.md
cp -r notes scripts/notes-copy
mv notes/hello.md notes/intro.md          # rename (same dir)
mv notes/intro.md scripts/                # move (different dir)
ls -R                                     # recursive listing

08Clean up — carefully

Always ls what you're about to delete, then delete:

ls scripts/notes-copy
rm -i scripts/notes-copy/letters.md       # -i = confirm each
rm -r scripts/notes-copy                  # remove the directory + contents
Danger — never type rm -rf ~ or rm -rf /, even as a joke. There is no undo.

09Capture your work

Save two artifacts inside ~/lab-01/:

cd ~/lab-01
history | tail -n 100 > transcript.txt    # the last 100 commands you ran
find . -type f -o -type d | sort > tree.txt
cat tree.txt

These two files are the deliverable.

Hints

  • Tab completion is your superpower. Press Tab once to complete; twice to list options.
  • Forgot sudo? sudo !! reruns the last command with sudo prepended.
  • pwd often. Three letters, one Enter — never wonder where you are.
  • Be paranoid about rm -rf. Habit-form ls first, rm -i when unsure.
  • Ctrl + R searches your history live. Try it.
  • Ctrl + C cancels the current command. Ctrl + Z suspends it (resume with fg).

Deliverable

Submit (paste into a doc, push to a repo, or commit alongside your homework):

  1. ~/lab-01/transcript.txt — your command history.
  2. ~/lab-01/tree.txt — the resulting directory tree.
  3. A one-paragraph reflection answering: which command surprised you, and which one will you reach for most often?

Stretch (optional)

  • Pipe + count: how many files (not directories) live in /etc? Build the answer with ls, grep, and wc -l.
  • Five biggest log files in /var/log: combine ls -lS with head.
  • script(1): start a recording with script ~/lab-01/session.log, do some commands, then exit. Open the log — see what got captured.
  • Watch a kernel buffer: dmesg | tail. What's the most recent thing the kernel logged?
