Course 01 — Linux Fundamentals — Syllabus

From "I've never opened a terminal" to "I can comfortably live in one." The first hands-on course of the program.

At a glance

Duration~2 weeks (8–12 hours of contact time + lab work)

PrerequisitesCourse 00 — DevOps Intro (recommended)

Overview

Students go from "I've never opened a terminal" to "I can stand up, operate, and troubleshoot a Linux box on my own." The course is taught in four parts: the shell (commands, paths, redirection, pipes, globbing); shell power-ups (aliases, history, line-editing, jobs, PATH, prompts, scripting basics); under the hood (kernel vs user space, system calls, sudo, boot, systemd, the FHS); and operating the system — package management, vi, permissions, users & groups, processes & resources, archives, scheduled tasks, storage and filesystems, LVM, logs, and writing your own systemd service. Every concept has a command to type, every lab has output to show.

Learning outcomes

By the end of this course, a student will be able to:

  • Navigate any Linux filesystem with pwd, cd, ls, pushd/popd, find, absolute vs relative paths.
  • Create, copy, move, delete, and symlink files and directories — including recursive operations with the right flags.
  • Read files (cat, less, head/tail -f) and shape streams with pipes, redirection (>, 2>, &>), and the text toolkit (grep, cut, sort, wc, tr, tee).
  • Use the help system (man, --help, info, whatis, apropos, man hier) without reaching for a search engine first.
  • Customize a shell environment: aliases, history expansion, PATH, prompt (PS1), and pick a shell intentionally.
  • Explain the kernel / user-space boundary, demonstrate it with strace, and use sudo responsibly.
  • Trace the boot process end-to-end (BIOS/UEFI → bootloader → kernel + initrd → init/systemd → login shell), read systemd targets, and inspect running services with systemctl & journalctl.
  • Identify all seven Linux file types by symbol, and navigate the FHS by purpose (/bin, /etc, /var, /usr, /proc, /sys, etc.).
  • Install, update, and remove software on both Debian/Ubuntu (apt + dpkg) and RHEL/Fedora (dnf + rpm); add a third-party repository with GPG verification.
  • Edit files in vi at survival level — modes, save/quit, search, basic editing.
  • Manage file permissions and ownership (chmod symbolic + octal, chown, SUID/SGID/sticky bit, umask) and manage users & groups (useradd, usermod, passwd, groupadd, /etc/passwd).
  • Inspect processes and system resources with top, ps, free, df, du; send signals with kill.
  • Bundle archives (tar, gzip, zstd) and schedule periodic tasks with cron or systemd timers.
  • Mount filesystems, edit /etc/fstab, and understand LVM (PV → VG → LV) at a conceptual level.
  • Read logs from /var/log and with journalctl — filter by time, severity, and service; recognise logrotate rules.
  • Write a custom .service systemd unit (paired with a .timer for periodic runs) and enable it on boot.

Deliverables

  • Homework labs — 5 labs:
    • lab-01-cli-tour — guided shell tour (navigate, create, view, delete).
    • lab-02-pipes-and-path — compose commands with pipes and globs; modify PATH.
    • lab-03-boot-and-fhs — inspect the boot timeline, list systemd services, walk the FHS.
    • lab-04-permissions-and-users — create users, set group ownership, walk SUID/SGID, run chmod scenarios.
    • lab-05-storage-and-services — mount a loopback device, edit /etc/fstab, write a .service + .timer, watch it fire.
  • Final project — Make a Linux Box Yours — provision a fresh Linux box, customize it end-to-end, add a user and a custom systemd service, mount a volume, build a system report with pipes, and ship the whole thing as a Git repo (~3–5 hr).

Resources

    • The Linux Command Line (William Shotts, free PDF) — excellent companion book.

Assessment

A student passes the course when they have:

  1. Completed all five homework labs (deliverables submitted as command transcripts + short reflections).
  2. Submitted the final project: a working Linux environment with the customizations described, plus a short write-up of the boot sequence and the systemd service they added.
  3. Demonstrated, in conversation, that they can answer: "what happens when I press Enter on ls -l /etc?" — touching shell, PATH, syscalls, and kernel.
