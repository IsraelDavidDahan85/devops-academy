Linux Fundamentals — Cheat Sheet

A one-document reference covering every command, flag, and shortcut from the course — across all four parts: the shell, power-ups, under the hood, and operating the system. Open in a browser, print to PDF, or keep it pinned in a tab. Slide numbers in parentheses point to the source.

Navigation (slide 5)

CommandWhat it does
pwdPrint working directory — where am I right now.
ls · ls -laList files; -l long format, -a include hidden, -h human-readable sizes.
cd <path>Change directory. cd ~ home, cd / root, cd - previous dir.
pushd / popdPush the current dir onto a stack and jump elsewhere; popd returns.
realpath <path>Resolve to an absolute path (follows symlinks).
Paths — anything starting with / is absolute. Everything else is relative to where you are right now.

File operations (slide 6)

CommandWhat it does
mkdir -p <path>Create directory; -p makes parents and is idempotent.
touch <file>Create an empty file (or bump modified time).
cp -r <src> <dst>Copy. -r recursive for directories, -p preserve mode/timestamp.
mv <src> <dst>Move or rename — same syntax for both.
rm -r <path>Remove. -r recursive, -i ask before each, -f force.
ln -s <target> <name>Create a symlink.
Danger — rm -rf does not ask twice. Never combine it with absolute paths you haven't verified — ls first, then delete.

The $PATH (slide 18)

A colon-separated list of directories the shell searches when you type a command. First match wins.

$ echo $PATH
/usr/local/bin:/usr/bin:/bin:/home/student/bin

$ which ls           # which one will actually run
/usr/bin/ls

# prepend your own bin, persist in ~/.bashrc:
export PATH="$HOME/bin:$PATH"
Careful — never delete /usr/bin from $PATH. You will lose access to ls, cat, rm — and the tools needed to fix it.

Permissions & ownership (slide 34)

The 9 permission characters split into three triplets: owner / group / other. Each is r read = 4, w write = 2, x execute = 1. Add them for octal.

CommandWhat it does
chmod 755 <f>rwxr-xr-x — owner all, others r+x (default for executables & dirs)
chmod 644 <f>rw-r--r-- (default for files)
chmod u+x <f>add execute to owner (symbolic)
chmod -R 644 <dir>recursive — every file inside
chown <u> <f>change owner
chown <u>:<g> <f>change owner and group together
chgrp <g> <f>change just the group
stat <f>full metadata — mode in both octal and symbolic

Writing your own systemd service (slide 43)

A .service file tells systemd how to run your program. Save it under /etc/systemd/system/, then reload.

# /etc/systemd/system/backup.service
[Unit]
Description=Nightly backup
After=network.target

[Service]
Type=oneshot
ExecStart=/usr/local/bin/backup.sh
User=backup
Restart=on-failure

[Install]
WantedBy=multi-user.target
CommandWhat it does
systemctl daemon-reloadreload after editing a unit file
systemctl enable <svc>start automatically on boot
systemctl start <svc>start now
systemctl status <svc>is it healthy? recent output?
journalctl -u <svc> -ffollow its logs
Pair with a timer — a backup.timer file (same name, .timer extension) runs the service on a schedule. See man systemd.timer.
