יסודות לינוקס — דף עזר

מסמך הפניה אחד המכסה כל פקודה, דגל וקיצור דרך מהקורס — בכל ארבעת חלקיו: ה-shell, שדרוגי shell, מתחת למכסה המנוע, ותפעול המערכת. פתחו בדפדפן, הדפיסו ל-PDF, או השאירו ב-tab פתוח. מספרי השקופיות בסוגריים מצביעים על המקור.

ניווט (שקופית 5)

פקודהמה היא עושה
pwdהדפסת תיקיית עבודה — איפה אני עכשיו.
ls · ls -laרשימת קבצים; -l פורמט ארוך, -a כולל מוסתרים, -h גדלים קריאים.
cd <path>שינוי תיקייה. cd ~ בית, cd / ראש, cd - תיקייה קודמת.
pushd / popdדחיפת התיקייה למחסנית וקפיצה למקום אחר; popd מחזיר.
realpath <path>פתרון לנתיב מוחלט (עוקב אחר symlinks).
נתיבים — כל מה שמתחיל ב-/ הוא מוחלט. כל השאר הוא יחסי למיקום הנוכחי שלכם.

פעולות על קבצים (שקופית 6)

פקודהמה היא עושה
mkdir -p <path>יצירת תיקייה; -p יוצר את ההורים ואידמפוטנטי.
touch <file>יצירת קובץ ריק (או עדכון חותמת הזמן).
cp -r <src> <dst>העתקה. -r רקורסיבי לתיקיות, -p שומר על הרשאות/זמן.
mv <src> <dst>העברה או שינוי שם — אותו תחביר לשניהם.
rm -r <path>מחיקה. -r רקורסיבי, -i אישור לכל אחד, -f מאלץ.
ln -s <target> <name>יצירת symlink.
סכנה — rm -rf לא שואל פעמיים. אף פעם אל תשלבו עם נתיבים מוחלטים שלא בדקתם — קודם ls, אחר כך delete.

ה-$PATH (שקופית 18)

רשימה מופרדת ב-colons של תיקיות שה-shell מחפש בהן כשמקלידים פקודה. ההתאמה הראשונה זוכה.

$ echo $PATH
/usr/local/bin:/usr/bin:/bin:/home/student/bin

$ which ls           # איזה אחד באמת ירוץ
/usr/bin/ls

# הוספת bin משלכם בהתחלה, ייצוב ב-~/.bashrc:
export PATH="$HOME/bin:$PATH"
בזהירות — אף פעם אל תמחקו את /usr/bin מ-$PATH. תאבדו גישה ל-ls, cat, rm — ולכלים שתצטרכו כדי לתקן.

הרשאות ובעלות (שקופית 34)

תשעת תווי ההרשאות מתחלקים לשלוש שלישיות: owner / group / other. כל אחד הוא r read = 4, w write = 2, x execute = 1. סכמו לערך אוקטלי.

פקודהמה היא עושה
chmod 755 <f>rwxr-xr-x — לבעלים הכל, לאחרים r+x (ברירת מחדל לקבצי הרצה ותיקיות)
chmod 644 <f>rw-r--r-- (ברירת מחדל לקבצים)
chmod u+x <f>הוספת הרשאת execute לבעלים (סימבולי)
chmod -R 644 <dir>רקורסיבי — כל קובץ בפנים
chown <u> <f>שינוי בעלים
chown <u>:<g> <f>שינוי בעלים וקבוצה יחד
chgrp <g> <f>שינוי קבוצה בלבד
stat <f>מטא-דאטה מלאה — mode באוקטלי ובסימבולי

כתיבת שירות systemd משלכם (שקופית 43)

קובץ .service מסביר ל-systemd איך להריץ את התוכנית שלכם. שמרו תחת /etc/systemd/system/ וטענו מחדש.

# /etc/systemd/system/backup.service
[Unit]
Description=Nightly backup
After=network.target

[Service]
Type=oneshot
ExecStart=/usr/local/bin/backup.sh
User=backup
Restart=on-failure

[Install]
WantedBy=multi-user.target
