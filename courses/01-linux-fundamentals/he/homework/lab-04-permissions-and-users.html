מעבדה 04 — הרשאות ומשתמשים

הקימו שני משתמשים אמיתיים, צרפו אותם לקבוצה משותפת, והשתמשו בהרשאות וב-bits מיוחדים כדי לבנות workspace שעובד נכון. אמתו את כל הצעדים עם ls -l, id ו-stat.

קורס01 — יסודות לינוקס
זמןכשעה
תוצר~/lab-04/transcript.txt + השורות של alice ו-bob ב-/etc/passwd + רפלקציה של פסקה
מצגות תואמות34–36 (חלק 04 — תפעול המערכת)

מטרה

בסיום המעבדה תוכלו ליצור משתמשים, לצרף אותם לקבוצות, לבנות תיקייה משותפת שמעניקה אוטומטית את הקבוצה הנכונה לקבצים חדשים באמצעות SGID, להגדיר chmod לשלושה תרחישים מהעולם האמיתי, ולהוכיח שה-sticky bit על /tmp עובד.

הקמה

נדרשת מכונת לינוקס אמיתית עם הרשאות sudo — VM, WSL2, או מכונה בענן. אל תשתמשו ב-container של Docker — חלק מהפקודות משנות את /etc/passwd ו-/etc/shadow, וההתנהגות בתוך container שונה ממערכת מלאה.

mkdir -p ~/lab-04
cd ~/lab-04
script -q transcript.txt        # התחלת הקלטה של כל מה שתעשו
# (הריצו :exit או Ctrl-D כשגמרתם)
טיפ — script(1) מקליט את כל הסשן ל-transcript.txt. הריצו את כל המשימות שלמטה בתוך ההקלטה, ואז צאו.

משימות

עבדו לפי הסדר. קראו כל שורה לפני שאתם לוחצים Enter.

01בדיקת המצב הנוכחי

whoami
id
cat /etc/passwd | head -5      # הפורמט: name:x:uid:gid:gecos:home:shell
cat /etc/group | head -5
ls -l /etc/shadow              # רק root יכול לקרוא

אתם אמורים לראות את המשתמש שלכם ב-/etc/passwd, ורשימת קבוצות מ-id.

02יצירת שני משתמשים

sudo useradd -m -s /bin/bash alice
sudo useradd -m -s /bin/bash bob
sudo passwd alice
sudo passwd bob

id alice
getent passwd alice
ls -ld /home/alice

אמתו שהמשתמשים קיימים, שיש להם תיקיות בית, ושהוקצו להם UIDs (כנראה 1001 ו-1002 במכונה חדשה).

03יצירת קבוצה משותפת

sudo groupadd devs
sudo usermod -aG devs alice
sudo usermod -aG devs bob

getent group devs              # אמור להציג את שני החברים
id alice                       # devs אמור להופיע בקבוצות של alice
ה--a קריטי. usermod -G devs alice (בלי -a) מחליף את רשימת הקבוצות של alice עם devs בלבד — היא תאבד את כל החברויות האחרות. תמיד usermod -aG.

04בניית workspace משותף עם SGID

sudo mkdir /srv/shared
sudo chown root:devs /srv/shared
sudo chmod 2775 /srv/shared        # 2 מקדים = SGID
ls -ld /srv/shared                 # ה-mode מציג 'rwxrwsr-x' — ה-'s' הוא SGID
stat /srv/shared

עכשיו תנו לכל משתמש להוסיף קובץ:

sudo -u alice touch /srv/shared/from-alice.txt
sudo -u bob   touch /srv/shared/from-bob.txt
ls -l /srv/shared

שני הקבצים אמורים להציג את קבוצה devs — זה ה-SGID בפעולה. בלעדיו, כל קובץ היה מקבל את הקבוצה הראשית של היוצר.

05שלושה תרחישי הרשאות

לכל תרחיש, הגדירו את ה-mode המתאים ואמתו עם ls -l.

תרחישמה אתם רוציםMode
סקריפט שכולם יכולים להריץבעלים הכל, אחרים r+x755
קובץ תצורה שרק root יכול לקרואבעלים r+w בלבד600
קובץ לוג שקבוצת devs יכולה להוסיף אליובעלים+קבוצה r+w, אחרים r664
 
# סקריפט שכולם יכולים להריץ
printf '#!/bin/bash\necho "hi from $USER"\n' | sudo tee /usr/local/bin/hi
sudo chmod 755 /usr/local/bin/hi
sudo -u alice /usr/local/bin/hi

# תצורה שרק root יכול לקרוא
sudo touch /etc/important.conf
sudo chmod 600 /etc/important.conf
ls -l /etc/important.conf
sudo -u alice cat /etc/important.conf    # אמור להיכשל ב-permission denied

# לוג משותף שניתן לכתיבה ע"י devs
sudo touch /srv/shared/devs.log
sudo chown root:devs /srv/shared/devs.log
sudo chmod 664 /srv/shared/devs.log
sudo -u alice sh -c 'echo "alice was here" >> /srv/shared/devs.log'
sudo -u bob   sh -c 'echo "bob was here"   >> /srv/shared/devs.log'
cat /srv/shared/devs.log

06הוכחת ה-sticky bit על /tmp

ls -ld /tmp                                # ה-mode מסתיים ב-'t' — sticky bit
sudo -u alice touch /tmp/alice.txt
sudo -u bob   rm /tmp/alice.txt            # אמור להיכשל — alice היא הבעלים

בלי sticky bit, כל מי שיש לו הרשאת write על /tmp יכול היה למחוק קבצים של אחרים. איתו — רק הבעלים (או root) יכול למחוק.

07בדיקה עם stat

stat /srv/shared
stat /usr/local/bin/hi
stat /tmp

stat מציג את ה-mode גם באוקטלי וגם בסימבולי — שימושי כשלא זוכרים מה זה מה.

08ניקוי (אופציונלי)

sudo userdel -r alice
sudo userdel -r bob
sudo groupdel devs
sudo rm -rf /srv/shared
sudo rm -f /etc/important.conf /usr/local/bin/hi

טיפים

  • לאחר הוספת משתמש לקבוצה, הוא צריך להתנתק ולהיכנס מחדש כדי שהקבוצה תיתן תוקף לסשן שלו. sudo -u alice ... עוקף את זה כי הוא מייצר סשן חדש.
  • getent passwd alice עובד גם כאשר מסד המשתמשים נמצא ב-LDAP או AD ולא ב-/etc/passwd. הרגילו את עצמכם להשתמש בו.
  • אם chmod 2775 נראה מוזר: ה-2 המקדים הוא SGID, ו-775 הוא ה-rwx הרגיל של owner/group/other. 4xxx זה SUID, 1xxx זה sticky.
  • ls -ld מציג את התיקייה עצמה ולא את התוכן שלה — שימושי לבדיקת mode על תיקייה.

תוצר

  1. ~/lab-04/transcript.txt — הקלטת ה-script המלאה שלכם.
  2. שורות /etc/passwd של alice ו-bob (שתי השורות האלה בלבד).
  3. פסקת רפלקציה אחת: מה ה-SGID bit על /srv/shared בעצם נותן? תארו מה היה משתבש בלעדיו.

אתגרים

  • תנו ל-alice sudo בלי סיסמה לפקודה אחת ספציפית (/usr/local/bin/hi) על ידי הנחת קובץ ב-/etc/sudoers.d/alice. בדקו עם sudo -n -u alice /usr/local/bin/hi.
  • השתמשו ב-setfacl כדי לתת ל-bob בלבד (לא לכל קבוצת devs) הרשאת write לקובץ ספציפי. אמתו עם getfacl.
  • הגדירו ACL ברירת מחדל על /srv/shared כך שכל קובץ חדש יהיה ניתן לכתיבה ע"י devs בלי קשר ל-umask: setfacl -d -m g:devs:rw /srv/shared.
