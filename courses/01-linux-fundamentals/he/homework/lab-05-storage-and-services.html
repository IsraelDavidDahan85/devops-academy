מעבדה 05 — אחסון ושירותים

צרו דיסק מדומה מקובץ רגיל, פרמטו אותו, מאונטו אותו, גרמו ל-mount לשרוד reboot, ואז כתבו שירות systemd + timer שיכתבו אליו לפי לוח זמנים. צפו בכל זה מתבצע דרך journalctl.

קורס01 — יסודות לינוקס
זמןכ-75 דקות
תוצר~/lab-05/transcript.txt + השורה שלכם ב-/etc/fstab + backup.service ו-backup.timer + פסקת רפלקציה
מצגות תואמות40–43 (חלק 04 — תפעול המערכת)

מטרה

בסיום המעבדה תוכלו: לבנות block device מקובץ רגיל (loopback), להניח עליו filesystem, למאונט אותו, לגרום לו לשרוד reboot דרך /etc/fstab עם UUID (אף פעם לא נתיב device), ולכתוב שירות systemd משלכם בשילוב timer שמריץ אותו לפי לוח זמנים.

הקמה

נדרשת מכונת לינוקס אמיתית עם sudo — VM, WSL2, או מכונה בענן. אל תשתמשו ב-Docker: גם loopback mounts וגם systemd מלא מתנהגים שונה ב-containers.

mkdir -p ~/lab-05
cd ~/lab-05
script -q transcript.txt        # התחלת הקלטה של כל מה שתעשו
# (הריצו :exit או Ctrl-D כשגמרתם)

משימות

01בדיקה של מה שכבר ממואנט

lsblk                          # block devices
findmnt                        # עץ של mounts
df -h                          # שטח פנוי לפי mount
cat /etc/fstab                 # מה ממואנט ב-boot

שימו לב למבנה: block device → partition אופציונלי → filesystem → mount point.

02בניית loopback device

Loopback device הוא קובץ רגיל שהקרנל מתייחס אליו כאל block device — מושלם לתרגול בלי דיסק נוסף.

# 256 MB של אפסים, ואז נגרום לקרנל לחבר loop device לקובץ
sudo dd if=/dev/zero of=/var/disks/lab.img bs=1M count=256 status=progress
sudo losetup -fP --show /var/disks/lab.img    # מדפיס /dev/loopN (זכרו את זה)
lsblk                                          # ה-device החדש אמור להופיע
מה קרה כאן — losetup -fP --show מוצא את מספר ה-loop הפנוי הבא (-f), מבקש מהקרנל לסרוק partitions (-P), ומדפיס את ה-device path: LOOP=$(sudo losetup -fP --show /var/disks/lab.img).

03פורמט עם filesystem

LOOP=$(sudo losetup -j /var/disks/lab.img | awk -F: '{print $1}')
echo "$LOOP"                                   # אישור

sudo mkfs.ext4 -L LAB "$LOOP"                  # ext4 עם label LAB
sudo blkid "$LOOP"                             # שמרו את ה-UUID

העתיקו את ה-UUID ש-blkid מדפיס — תזדקקו לו בשלב 5.

04Mount ידני

sudo mkdir -p /mnt/lab
sudo mount "$LOOP" /mnt/lab
findmnt /mnt/lab                               # אימות
df -h /mnt/lab                                 # שטח פנוי

# כתבו משהו
echo "hello from $(date)" | sudo tee /mnt/lab/hello.txt
ls -l /mnt/lab

05שמירת ה-mount דרך /etc/fstab

נתיב כמו /dev/loop0 יכול להשתנות בין boots — אבל UUID לעולם לא. תמיד מאונטו לפי UUID ב-/etc/fstab.

UUID=$(sudo blkid -s UUID -o value "$LOOP")
echo "$UUID"

# הוסיפו את השורה ל-fstab (אמתו את ה-UUID לפני!)
echo "UUID=$UUID  /mnt/lab  ext4  loop,defaults,nofail  0  2" | sudo tee -a /etc/fstab

# מאונטו מחדש את הכל מ-fstab כדי לאמת
sudo umount /mnt/lab
sudo mount -a                                  # מאונטים את כל מה ש-fstab מגדיר
findmnt /mnt/lab                               # חזר, ממואנט דרך fstab
האפשרות nofail חשובה. בלעדיה, דיסק חסר או שבור ב-boot יזרוק את המערכת למצב emergency rescue. nofail אומר "אם ה-mount הזה נכשל, המשיכו ב-boot". לדיסקים לא קריטיים (הכל חוץ מ-root), תמיד nofail.

06כתיבת שירות systemd

צרו את /usr/local/bin/lab-tick.sh — הסקריפט שהשירות יריץ:

sudo tee /usr/local/bin/lab-tick.sh >/dev/null <<'EOF'
#!/bin/bash
set -euo pipefail
echo "[$(date --iso-8601=seconds)] tick on $(hostname)" >> /mnt/lab/tick.log
EOF
sudo chmod +x /usr/local/bin/lab-tick.sh
sudo /usr/local/bin/lab-tick.sh                # בדיקת עשן
cat /mnt/lab/tick.log

כעת כתבו את ה-service unit ב-/etc/systemd/system/lab-tick.service:

sudo tee /etc/systemd/system/lab-tick.service >/dev/null <<'EOF'
[Unit]
Description=Lab 05 — write one tick into the lab disk

[Service]
Type=oneshot
ExecStart=/usr/local/bin/lab-tick.sh
EOF

07שילוב עם timer

וה-timer ב-/etc/systemd/system/lab-tick.timer — אותו שם, סיומת .timer. systemd מקשר אותם אוטומטית.

sudo tee /etc/systemd/system/lab-tick.timer >/dev/null <<'EOF'
[Unit]
Description=Run lab-tick every minute

[Timer]
OnBootSec=30s
OnUnitActiveSec=1min
AccuracySec=1s
Persistent=true

[Install]
WantedBy=timers.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable --now lab-tick.timer
systemctl list-timers --all | grep lab-tick     # מתי הוא מופעל בפעם הבאה

08צפייה בפעולה

journalctl -u lab-tick.service -f               # מעקב חי; Ctrl-C לעצירה
# חכו כ-2 דקות
cat /mnt/lab/tick.log                           # רשומה כל דקה

09בדיקת ה-unit והפעילות האחרונה

systemctl status lab-tick.service
systemctl status lab-tick.timer
systemctl cat lab-tick.service                  # מציג את קובץ ה-unit שממנו הוא רץ
journalctl -u lab-tick.service --since "10 min ago"

10ניקוי (אופציונלי אך מומלץ)

sudo systemctl disable --now lab-tick.timer
sudo rm /etc/systemd/system/lab-tick.{service,timer}
sudo systemctl daemon-reload

sudo umount /mnt/lab
sudo sed -i.bak "/$UUID/d" /etc/fstab           # הסרת השורה מ-fstab
sudo losetup -d "$LOOP"
sudo rm /var/disks/lab.img /usr/local/bin/lab-tick.sh
sudo rmdir /mnt/lab

טיפים

  • תמיד מאונטו לפי UUID (או LABEL) ב-/etc/fstab, לעולם לא לפי device path. מספרי device מתחלפים בין boots.
  • אם mount -a נכשל, גם ה-boot ייכשל. בדקו לפני שעושים reboot — זו בדיוק הסיבה שמריצים את זה ידנית קודם.
  • systemctl daemon-reload נדרש אחרי כל עריכה של unit file. בלעדיו, השינוי החדש לא יחול והקונפיג הישן ימשיך לפעול בשקט.
  • השם של קובץ ה-timer חייב להתאים לקובץ ה-service — foo.timer מפעיל את foo.service כברירת מחדל. לשירות אחר, הוסיפו Unit=other.service תחת [Timer].
  • Persistent=true אומר ל-systemd: אם ה-timer היה אמור להפעיל בזמן שהמכונה כבויה, הריצו פעם אחת ב-boot כדי להשלים.

תוצר

  1. ~/lab-05/transcript.txt — הקלטת הסשן המלאה שלכם.
  2. השורה המדויקת שהוספתם ל-/etc/fstab.
  3. התוכן של /etc/systemd/system/lab-tick.service ו-lab-tick.timer.
  4. פסקת רפלקציה אחת: למה מאונטים לפי UUID במקום device path? למה קיים פיצול בין timer ל-service — האם cron לא היה עושה את אותו דבר?

אתגרים

  • שנו את ה-timer לשימוש ב-calendar expression (OnCalendar=*:0/5 — כל 5 דקות לפי שעון הקיר) במקום OnUnitActiveSec. אמתו עם systemd-analyze calendar "*:0/5".
  • הוסיפו RandomizedDelaySec=10s תחת [Timer] וצפו איך זמני ההפעלה האמיתיים מתפזרים.
  • הגדילו את ה-loopback: עצרו את ה-timer, truncate -s 512M /var/disks/lab.img, losetup -c "$LOOP", resize2fs "$LOOP", מאונטו מחדש, אמתו עם df -h /mnt/lab.
  • גרמו לשירות לרוץ כמשתמש לא-root. הוסיפו User=youruser ב-[Service] ועדכנו chown על ה-mount בהתאם.
