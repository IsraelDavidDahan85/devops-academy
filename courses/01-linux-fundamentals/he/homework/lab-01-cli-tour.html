מעבדה 01 — סיור CLI

לנוע במערכת קבצים של לינוקס בביטחון — לנווט, ליצור, לבדוק, להעתיק ולנקות, הכל מתוך ה-shell.

קורס01 — יסודות לינוקס
זמןכ-45 דקות
תוצר~/lab-01/transcript.txt + ~/lab-01/tree.txt + רפלקציה של פסקה אחת
תואם למצגתשקופיות 4–9 (חלק 01 — ה-Shell)

מטרה

לשבת מול כל מכונת לינוקס ולחוש בנוח תוך פחות מ-30 שניות. בסיום המעבדה תוכלו לנווט, ליצור קבצים, לצפות בהם, להעתיק/לשנות שם, ולנקות בבטחה — בלי להגיע למנוע חיפוש.

הכנה

אתם צריכים טרמינל עובד בסביבת לינוקס אמיתית:

  • VM של לינוקס (multipass, VirtualBox, UTM, Lima וכו׳),
  • WSL2 על Windows,
  • VM בענן עם SSH, או
  • macOS — רוב הפקודות עובדות, אבל כמה דגלים שונים (במיוחד ls, tail ו-stat).

פתחו טרמינל. ודאו שאתם בתיקיית הבית:

echo $HOME
pwd

אם pwd לא תואם ל-$HOME, הריצו cd ~ קודם.

משימות

עברו עליהן בסדר. אל תדביקו עיוור — קראו כל שורה לפני שאתם לוחצים Enter, במיוחד את ההרסניות.

01התמצאות

whoami            # תחת איזה משתמש אני מחובר
hostname          # על איזו מכונה
pwd               # איפה אני
ls                # מה יש כאן
ls -la            # כל האמת — כולל קבצים מוסתרים

עכשיו תציצו במערכת עצמה — בלי לשנות כלום:

ls /              # ראש מערכת הקבצים
ls /etc | head    # קבצי תצורה
ls /var/log       # קבצי לוג
שאלות לתשובה עצמית: כמה תיקיות יש ב-/? בערך כמה קבצים יש ב-/etc?

02בניית סביבת עבודה

יצירת מבנה מקונן בפעולה אחת:

mkdir -p ~/lab-01/{notes,scripts,logs}
cd ~/lab-01
ls -la

אתם אמורים לראות שלוש תיקיות. mkdir -p הוא אידמפוטנטי — הרצה חוזרת לא תזיק.

03יצירת קבצים

שלוש דרכים ליצור קובץ:

touch notes/hello.md                              # קובץ ריק
echo "first line"  > notes/journal.md             # דריסה (קובץ עם שורה אחת)
echo "second line" >> notes/journal.md            # הוספה לסוף
printf "alpha\nbeta\ngamma\n" > notes/letters.md  # מולטי-שורה

אישור:

ls -la notes/
cat notes/journal.md
cat notes/letters.md

04צפייה בקבצים

נסו את כל הקוראים על אותו קובץ:

cat notes/letters.md
head -n 2 notes/letters.md
tail -n 2 notes/letters.md
less notes/letters.md      # q לסיום; /alpha ואז n/N לחיפוש

עכשיו נסו את הקובץ החי. בטרמינל A:

tail -f logs/live.log

בטרמינל שני (פתחו טאב חדש):

cd ~/lab-01
for i in 1 2 3; do
  echo "event $i" >> logs/live.log
  sleep 1
done

תסתכלו על טרמינל A. השורות החדשות מופיעות כשהן נכתבות. ככה מהנדסים צופים בלוגים בייצור. לחצו Ctrl + C בטרמינל A כשתסיימו.

05רשימת קבצים כמו מקצוען

ls -l notes/                # פורמט ארוך
ls -lah notes/              # + גדלים קריאים, + מוסתרים
ls -ltr notes/              # ממוין לפי זמן, הישן ראשון
ls -lSr notes/              # ממוין לפי גודל, הקטן ראשון

קראו את העמודות של ls -l בקפידה — הרשאות, links, בעלים, קבוצה, גודל, תאריך, שם. (שקופית 08 כוללת את דיאגרמת האנטומיה.)

06שימוש במערכת העזרה

בלי לחפש באינטרנט, מצאו:

  • הדגל של cp ששומר הרשאות וחותמות זמן.
  • הדגל של rm ששואל לפני כל מחיקה.
  • מה tee עושה.

השתמשו בכלים הבאים:

man cp           # מדריך מלא; q לסיום
cp --help        # הפניה מהירה
whatis tee       # תיאור של שורה אחת
type ls          # האם זה binary, alias, או builtin
which ls         # איפה ב-$PATH הפקודה יושבת

07העתקה, שינוי שם והעברה

cp notes/letters.md notes/letters-backup.md
cp -r notes scripts/notes-copy
mv notes/hello.md notes/intro.md          # שינוי שם (אותה תיקייה)
mv notes/intro.md scripts/                # העברה (תיקייה אחרת)
ls -R                                     # רשימה רקורסיבית

08ניקיון — בזהירות

תמיד תריצו ls על מה שאתם עומדים למחוק, ורק אז תמחקו:

ls scripts/notes-copy
rm -i scripts/notes-copy/letters.md       # -i = אישור לכל אחד
rm -r scripts/notes-copy                  # מחיקת התיקייה + תוכן
סכנה — אל תקלידו rm -rf ~ או rm -rf /, אפילו לא בצחוק. אין undo.

09שמירת העבודה

שמרו שני תוצרים בתוך ~/lab-01/:

cd ~/lab-01
history | tail -n 100 > transcript.txt    # 100 הפקודות האחרונות שלכם
find . -type f -o -type d | sort > tree.txt
cat tree.txt

שני הקבצים האלה הם התוצר.

טיפים

  • השלמה אוטומטית היא מעצמת-העל שלכם. לחצו Tab פעם אחת להשלמה; פעמיים לרשימת אפשרויות.
  • שכחתם sudo? sudo !! מריץ מחדש את הפקודה האחרונה עם sudo.
  • pwd לעיתים קרובות. שלוש אותיות, Enter אחד — אל תהיו במצב של אי-ידיעה איפה אתם.
  • תהיו פרנואידים לגבי rm -rf. תהפכו ל-ls ראשון להרגל, rm -i כשלא בטוחים.
  • Ctrl + R מחפש בהיסטוריה בזמן אמת. נסו.
  • Ctrl + C מבטל את הפקודה הנוכחית. Ctrl + Z משעה אותה (להמשך — fg).

תוצר

הגישו (הדביקו במסמך, דחפו לריפו, או commit כחלק משיעורי הבית):

  1. ~/lab-01/transcript.txt — היסטוריית הפקודות שלכם.
  2. ~/lab-01/tree.txt — מבנה התיקיות שנוצר.
  3. רפלקציה של פסקה אחת: איזו פקודה הפתיעה אתכם, ובאיזו תשתמשו הכי הרבה?

אתגרים (אופציונלי)

  • Pipe + ספירה: כמה קבצים (לא תיקיות) יש ב-/etc? בנו את התשובה עם ls, grep ו-wc -l.
  • חמשת קבצי הלוג הגדולים ב-/var/log: שלבו ls -lS עם head.
  • script(1): התחילו הקלטה עם script ~/lab-01/session.log, עשו כמה פקודות, ואז exit. פתחו את הלוג — מה נתפס.
  • תסתכלו על מאגר הקרנל: dmesg | tail. מה הדבר האחרון שהקרנל רשם?
