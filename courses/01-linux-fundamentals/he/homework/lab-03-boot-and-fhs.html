מעבדה 03 — Boot ו-FHS

להציץ מתחת למכסה המנוע — לעקוב אחר לוח הזמנים של ה-boot עם dmesg ו-journalctl, לבדוק targets ושירותים של systemd, לזהות את כל שבעת סוגי הקבצים בלינוקס, ולסייר בתיקיות הראשיות של FHS.

קורס01 — יסודות לינוקס
זמןכשעה
תוצר~/lab-03/transcript.txt + ~/lab-03/boot-timeline.md + רפלקציה של פסקה אחת
תואם למצגתשקופיות 20–22 (תהליך boot · systemd targets · סוגי קבצים ו-FHS)
למשתמשי macOS — המעבדה הזו דורשת לינוקס אמיתי. רוב הפקודות (journalctl, systemctl, /proc) לא קיימות ב-macOS. השתמשו ב-VM, WSL2, או SSH לשרת לינוקס בענן.

מטרה

להפסיק לראות את מערכת ההפעלה כקופסה שחורה. בסוף המעבדה תוכלו: לעקוב אחר מה שקורה בין הדלקה ל-login shell; לזהות מה רץ ולמה; להבחין בין קובץ רגיל ל-device, socket, או named pipe במבט אחד; ולנווט בכל מערכת קבצים לא מוכרת של לינוקס לפי תפקיד ולא לפי זיכרון.

הכנה

סביבת לינוקס אמיתית עם systemd (Ubuntu, Debian, Fedora, Arch — כמעט הכל המודרני). אשרו:

uname -a               # קרנל + ארכיטקטורה
cat /etc/os-release    # זהות הדיסטרו
systemctl --version    # אשרו ש-systemd הוא ה-init

צרו סביבת עבודה למעבדה:

mkdir -p ~/lab-03
cd ~/lab-03

משימות

01הלוגים של הקרנל עצמו

dmesg מדפיס את ה-ring buffer של הקרנל — מה שהקרנל עצמו רשם מההדלקה. בחלק מהדיסטרוס דרוש sudo.

dmesg | head -n 20          # השורות הראשונות אחרי boot
dmesg | tail -n 20          # האירועים העדכניים של הקרנל
dmesg | grep -i "usb"       # אילו התקני USB הקרנל ראה
dmesg | grep -i "ext4\|xfs\|btrfs"   # mounts של מערכות קבצים
מה הדבר הראשון שהקרנל שלכם רשם אחרי boot? איזה CPU הוא זיהה?

02היומן מה-boot הנוכחי

journalctl הוא יומן ה-systemd המאוחד. -b = boot הנוכחי, -b -1 = boot קודם.

journalctl -b | head -n 30         # boot מההתחלה
journalctl -b --no-pager | wc -l   # כמה שורות log ב-boot הזה
journalctl -b -p err               # רק שגיאות
journalctl --list-boots            # כל boot שתועד
journalctl _PID=1 -b | head -n 20  # רק מה ש-PID 1 (systemd) אמר

03systemd targets ושירותים

Targets הם קבוצות שירותים מאוגדות בשם. שירותים הם units ש-systemd מנהל.

systemctl get-default                       # multi-user.target? graphical.target?
systemctl list-units --type=target          # כל ה-targets הפעילים
systemctl list-units --type=service --state=running | head -n 20
systemctl list-units --failed               # משהו נשבר?
systemctl status                             # עץ התמונה המלאה

בחרו שירות אחד שרץ על המכונה שלכם (למשל ssh.service, cron.service, NetworkManager.service) ובדקו אותו:

systemctl status <service>.service
systemctl cat <service>.service              # קובץ ה-unit עצמו
journalctl -u <service>.service -n 30        # הלוגים שלו

04זיהוי שבעת סוגי הקבצים

התו הראשון ב-ls -l אומר לכם איזה סוג קובץ אתם מסתכלים עליו: - רגיל, d תיקייה, l symlink, c char device, b block device, p named pipe, s socket.

מצאו דוגמה אחת לכל אחד:

ls -l /etc/hosts                       # -    קובץ רגיל
ls -ld /var/log                        # d    תיקייה
ls -l /bin                             # l    symlink (בהרבה דיסטרוס /bin → /usr/bin)
ls -l /dev/tty                         # c    character device
ls -l /dev/sda 2>/dev/null || ls -l /dev/disk0   # b    block device
ls -l /dev/log 2>/dev/null || find /run -type p 2>/dev/null | head -n1 | xargs -I{} ls -l {}
ls -l /var/run/docker.sock 2>/dev/null || find /run -type s 2>/dev/null | head -n1 | xargs -I{} ls -l {}
תיעדו דוגמה אחת לכל סוג — הדביקו אותן לסקציה ב-boot-timeline.md אחר כך.

05FHS — סיור הליכה

בקרו בכל תיקייה ראשית והסתכלו על דבר אחד מעיד.

ls /                       # הראש המלא
ls /bin | wc -l            # כמה binaries של משתמש
ls /sbin | head            # binaries לניהול מערכת
cat /etc/os-release        # /etc = תצורה
ls /home                   # תיקייה לכל משתמש
ls /var/log | head         # /var = נתונים משתנים (בעיקר logs)
ls /usr/share | head       # /usr = תוכנות מותקנות, read-only
ls /tmp                    # /tmp = scratch (נמחק בדרך כלל ב-reboot)
cat /proc/cpuinfo | head   # /proc = תצוגה חיה של הקרנל
cat /proc/meminfo | head
ls /sys/class | head       # /sys = אובייקטים של הקרנל (devices, modules)
ls /dev | head             # /dev = קבצי התקנים

06הצצה ל-PID 1

כל תהליך בלינוקס יורד מ-PID 1. במכונות systemd, זה systemd עצמו.

ps -p 1 -o pid,comm,args
ls -la /proc/1/exe         # symlink ל-binary שרץ
cat /proc/1/cmdline | tr '\0' ' '; echo  # איך הוא הופעל
ls /proc/1/fd | head       # file descriptors ש-PID 1 פתח

07כתבו את לוח הזמנים של ה-boot שלכם

פתחו את ~/lab-03/boot-timeline.md בעורך וכתבו נרטיב קצר: "במכונה שלי, הנה מה שקרה בין החשמל ל-login." השתמשו בראיות האמיתיות שאספתם.

# התחילו את הקובץ עם תבנית:

08שמירת העבודה

cd ~/lab-03
history | tail -n 150 > transcript.txt
{
  echo "=== uname ===";        uname -a
  echo "=== os-release ===";   cat /etc/os-release
  echo "=== default target ==="; systemctl get-default
  echo "=== failed units ==="; systemctl list-units --failed
} >> transcript.txt

טיפים

  • journalctl -f עוקב אחרי רשומות log חדשות בזמן אמת — כמו tail -f ליומן המערכת. שימושי לראות מה קורה כשאתם start/stop שירותים בטרמינל אחר.
  • אל תריצו isolate בקלות. systemctl isolate rescue.target מבעיט כל שירות אחר — בסדר ב-VM אישי, רע ב-SSH לשרת שאכפת לכם ממנו.
  • /proc הוא מזויף. הקבצים שם לא יושבים על דיסק — הם תצוגה חיה של נתוני קרנל. לכן cat /proc/cpuinfo תמיד מציג מידע עכשווי.
  • סוג הקובץ l נמצא בכל מקום. בהרבה דיסטרוס /bin, /sbin, /lib הם symlinks ל-/usr. עולם ה-"usrmerge".
  • אם פקודה דורשת sudo ואין לכם, תרשמו את זה ב-transcript ("dmesg דורש sudo ב-Ubuntu >= 23"). המעבדה עדיין נחשבת.

תוצר

הגישו:

  1. ~/lab-03/transcript.txt — היסטוריית פקודות + uname + os-release + default target + failed units.
  2. ~/lab-03/boot-timeline.md — הנרטיב המלא שלכם של ששת שלבי ה-boot, ועוד שבע דוגמאות של סוגי קבצים.
  3. רפלקציה של פסקה אחת: איזה שלב שינה לכם את התפיסה של "מה זו מערכת הפעלה"?

אתגרים (אופציונלי)

  • עץ תלויות שירותים: הריצו systemctl list-dependencies multi-user.target (או ה-default target שלכם). מה בשורש?
  • זמן boot: systemd-analyze אומר כמה זמן boot לקח. systemd-analyze blame מסדר שירותים לפי זמן הפעלה. איזה שירות הכי איטי במכונה שלכם?
  • Bind mounts ב-/proc: השוו cat /proc/self/mounts ל-mount. מה ההבדל?
  • כתבו unit משלכם ל-systemd (אל תפעילו אותו). הניחו קובץ .service ב-~/.config/systemd/user/ שמריץ echo hello >> ~/lab-03/hello.log. הריצו עם systemctl --user start <name>.service. קראו את היומן שלו.
