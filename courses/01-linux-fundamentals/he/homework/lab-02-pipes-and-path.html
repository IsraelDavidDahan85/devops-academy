מעבדה 02 — Pipes ו-PATH

להרכיב פקודות עם pipes, הפניות ו-globbing — ואז לשלוט איפה ה-shell מוצא תוכניות על ידי עריכת $PATH.

קורס01 — יסודות לינוקס
זמןכ-45 דקות
תוצר~/lab-02/transcript.txt + ~/bin/greet (סקריפט מותאם על ה-PATH) + רפלקציה של פסקה אחת
תואם למצגתשקופיות 14–16 (Pipes ו-globs · PATH · Prompt)

מטרה

להפסיק להתייחס לפקודות ככלים מבודדים. ללמוד להרכיב — להעביר את הפלט של פקודה אחת לקלט של אחרת, להפנות זרמים לקבצים, ולהשתמש בתווי wildcard כדי לפעול על הרבה קבצים בו-זמנית. ואז לקחת שליטה על הסביבה שלכם דרך עריכת $PATH כדי שה-shell ימצא את הסקריפטים שלכם.

הכנה

טרמינל לינוקס (VM, WSL, ענן או macOS — רוב הדגלים תואמים). ודאו שאתם בבית:

cd ~
pwd

צרו סביבת עבודה למעבדה הזו:

mkdir -p ~/lab-02/{logs,reports}
cd ~/lab-02

משימות

01בדיקת $PATH

לפני שתשנו את $PATH, תסתכלו מה כבר שם.

echo $PATH
echo $PATH | tr ':' '\n'      # תיקייה אחת בכל שורה
which ls
which python3 || echo "אין python3 ב-PATH"
type cd                       # builtin? alias? קובץ?
איזו תיקייה היא הראשונה ב-$PATH שלכם? כשאתם מקלידים ls, למה /usr/bin/ls רץ ולא /bin/ls?

02Globbing — תווי wildcard

צרו כמה קבצים להתנסות:

cd ~/lab-02
touch alpha.txt beta.txt gamma.txt readme.md
touch logs/{2024-01,2024-02,2024-03}.log

נסו כל wildcard:

ls *.txt                # כל קבצי .txt
ls *.md                 # כל קבצי .md
ls logs/*.log           # כל קבצי .log ב-logs/
ls logs/2024-0?.log     # ? = תו אחד בדיוק
ls [ab]*.txt            # מתחיל ב-a או b
ls [!a]*.txt            # לא מתחיל ב-a

03הפניות — > · >> · 2>

זרמים: stdout (1), stderr (2). הפנו אותם בנפרד.

ls -la           >  reports/listing.txt    # stdout דריסה
date             >> reports/listing.txt    # stdout הוספה
ls /nope         2> reports/errors.txt     # stderr לקובץ
ls / /nope       >  reports/mixed.txt 2>&1  # שניהם לאותו קובץ
cat reports/listing.txt
מה יש ב-reports/errors.txt? מה ההבדל בין 2> ל->?

04Pipes — הרכבת פקודות

ה-pipe (|) שולח את stdout של הפקודה השמאלית כ-stdin לימנית. קראו כל דוגמה והריצו:

# ספרו כמה קבצי .log יש ב-/var/log
ls /var/log/*.log 2>/dev/null | wc -l

# 5 הקבצים הגדולים ב-/etc
ls -lS /etc 2>/dev/null | head -n 6

# מי שמריץ bash עכשיו
ps aux | grep bash | grep -v grep

# Shells ייחודיים על המערכת
cat /etc/passwd | cut -d: -f7 | sort -u

05בנו pipeline משלכם

שלבו pipes + globs + הפניות. שמרו את הפלט לקובץ.

# ספרו שורות error בכל הלוגים של המעבדה
echo "error_1" >  logs/2024-01.log
echo "info"    >> logs/2024-01.log
echo "error_2" >  logs/2024-02.log
echo "ERROR_3" >  logs/2024-03.log

cat logs/*.log | grep -i "error" | sort -u > reports/all-errors.txt
cat reports/all-errors.txt
wc -l reports/all-errors.txt

06צרו פקודה משלכם, הוסיפו ל-$PATH

צרו סקריפט קצר שמברך לפי שם.

mkdir -p ~/bin
cat > ~/bin/greet <<'EOF'
#!/usr/bin/env bash
name="${1:-friend}"
echo "Hello, $name. Welcome to lab-02."
EOF
chmod +x ~/bin/greet

נסו להריץ. עדיין לא יעבוד:

greet world
# bash: greet: command not found

זה כי ~/bin לא ב-$PATH. הוסיפו לסשן הזה:

export PATH="$HOME/bin:$PATH"
echo $PATH | head -c 60        # ודאו ש-~/bin ראשון
greet world                    # → Hello, world. ...

07קיבעו את השינוי

export חי רק לסשן הנוכחי. כדי להפוך אותו לקבוע:

# משתמשי bash:
echo 'export PATH="$HOME/bin:$PATH"' >> ~/.bashrc

# משתמשי zsh:
echo 'export PATH="$HOME/bin:$PATH"' >> ~/.zshrc

# טעינה מחדש בלי לצאת:
source ~/.bashrc   # או source ~/.zshrc

פתחו טרמינל חדש והקלידו greet $(whoami) — אמור לעבוד בלי export מחדש.

08שמירת העבודה

cd ~/lab-02
history | tail -n 100 > transcript.txt
ls -laR ~/lab-02 ~/bin/greet >> transcript.txt
echo "--- PATH ---"          >> transcript.txt
echo $PATH                    >> transcript.txt

טיפים

  • 2>/dev/null משתיק את stderr — שימושי כשפקודה ממלאת את המסך ב-"Permission denied" עבור קבצים שלא מעניינים אתכם.
  • &> ב-bash מפנה גם stdout וגם stderr בו-זמנית. פחות הקלדה מאשר > file 2>&1.
  • type <cmd> אומר לכם אם פקודה היא binary ב-PATH, alias, builtin של shell, או פונקציה. שלפו אותו כש"מאיפה זה בא?" חשוב.
  • קבצי rc של shell: bash קורא ~/.bashrc ל-interactive shells, ~/.bash_profile ל-login shells; zsh קורא ~/.zshrc. אם השינויים לא תופסים, ייתכן שאתם עורכים את הקובץ הלא נכון.
  • סדר חשוב ב-$PATH: התיקייה הראשונה שמתאימה זוכה. שימו ~/bin בהתחלה כדי לדרוס פקודות מערכת.

תוצר

הגישו:

  1. ~/lab-02/transcript.txt — היסטוריית הפקודות + ls + dump של PATH.
  2. ~/bin/greet — הסקריפט שלכם (שורה אחת של "יצרתי את זה" מספיקה כתוצר).
  3. השורה המדויקת שהוספתם ל-~/.bashrc או ~/.zshrc.
  4. רפלקציה של פסקה אחת: איזו טכניקת הרכבה (pipe מול redirect מול glob) הרגישה הכי טבעית, ומה הפתיע אתכם?

אתגרים (אופציונלי)

  • אתגר one-liner: באמצעות pipeline אחד בלבד (בלי קבצים זמניים), הדפיסו את 3 ה-shells הכי נפוצים במערכת.
  • ארכיאולוגיית PATH: עבור כל תיקייה ב-$PATH, ספרו כמה executables היא מכילה. רמז: ls -la <dir> | wc -l בלולאה, או find <dir> -maxdepth 1 -executable.
  • tee: השתמשו ב-tee כדי לכתוב פלט של pipeline לקובץ וגם ל-stdout בו-זמנית. שלבו עם ה-pipeline של errors ממשימה 05.
  • וריאציות shebang: שכתבו את greet עם #!/bin/sh. האם משהו נשבר? למה?
