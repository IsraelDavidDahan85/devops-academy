קורס 01 — יסודות לינוקס — סילבוס

מ"מעולם לא פתחתי טרמינל" ל"אני יכול לחיות בתוך טרמינל בנוחות". הקורס המעשי הראשון בתוכנית.

במבט אחד

משךכשבועיים (8–12 שעות פרונטליות + עבודת מעבדה)
רמהמתחילים — מניחים שאתם יודעים להשתמש במחשב אך לא בטרמינל
דרישות קדםקורס 00 — מבוא ל-DevOps (מומלץ)

סקירה

הסטודנטים עוברים מ"מעולם לא פתחתי טרמינל" ל"אני יכול להקים, להפעיל ולנפות שגיאות במכונת לינוקס לבד". הקורס מחולק לארבעה חלקים: ה-shell (פקודות, נתיבים, הפניות, pipes, globbing); שדרוגי shell (aliases, היסטוריה, עריכת שורה, jobs, PATH, prompts, יסודות scripting); מתחת למכסה המנוע (קרנל מול מרחב המשתמש, system calls, sudo, boot, systemd, ה-FHS); ותפעול המערכת — ניהול חבילות, vi, הרשאות, משתמשים וקבוצות, תהליכים ומשאבים, ארכיונים, משימות מתוזמנות, אחסון ומערכות קבצים, LVM, לוגים, וכתיבת שירות systemd משלכם. לכל מושג יש פקודה להקליד, לכל מעבדה יש פלט להציג.

תוצרי למידה

בסיום הקורס, סטודנט יוכל:

  • לנווט בכל מערכת קבצים של לינוקס באמצעות pwd, cd, ls, pushd/popd, find, ונתיבים מוחלטים מול יחסיים.
  • ליצור, להעתיק, להעביר, למחוק וליצור symlinks לקבצים ותיקיות — כולל פעולות רקורסיביות עם הדגלים הנכונים.
  • לקרוא קבצים (cat, less, head/tail -f) ולעבד streams עם pipes, הפניות (>, 2>, &>) וערכת הכלים לטקסט (grep, cut, sort, wc, tr, tee).
  • להשתמש במערכת העזרה (man, --help, info, whatis, apropos, man hier) בלי להגיע למנוע חיפוש קודם.
  • להתאים אישית סביבת shell: aliases, היסטוריה, PATH, prompt (PS1), ולבחור shell במודע.
  • להסביר את הגבול בין הקרנל למרחב המשתמש, להדגים אותו עם strace, ולהשתמש ב-sudo באחריות.
  • לעקוב אחר תהליך ה-boot מקצה לקצה (BIOS/UEFI → bootloader → קרנל + initrd → init/systemd → login shell), לקרוא systemd targets, ולבדוק שירותים פעילים עם systemctl ו-journalctl.
  • לזהות את כל שבעת סוגי הקבצים בלינוקס לפי הסמל, ולנווט ב-FHS לפי תפקיד (/bin, /etc, /var, /usr, /proc, /sys, וכו׳).
  • להתקין, לעדכן ולהסיר תוכנה גם ב-Debian/Ubuntu (apt + dpkg) וגם ב-RHEL/Fedora (dnf + rpm); להוסיף repo חיצוני עם אימות GPG.
  • לערוך קבצים ב-vi ברמת הישרדות — modes, שמירה/יציאה, חיפוש, עריכה בסיסית.
  • לנהל הרשאות ובעלות על קבצים (chmod סימבולי + אוקטלי, chown, SUID/SGID/sticky, umask) ולנהל משתמשים וקבוצות (useradd, usermod, passwd, groupadd, /etc/passwd).
  • לבדוק תהליכים ומשאבי מערכת עם top, ps, free, df, du; ולשלוח signals עם kill.
  • לארוז ארכיונים (tar, gzip, zstd) ולתזמן משימות תקופתיות עם cron או timers של systemd.
  • למאונט מערכות קבצים, לערוך את /etc/fstab, ולהבין LVM (PV → VG → LV) ברמת מושג.
  • לקרוא לוגים מ-/var/log ועם journalctl — לסנן לפי זמן, חומרה ושירות; להכיר את logrotate.
  • לכתוב unit מותאם של systemd (.service) — בשילוב .timer להרצה תקופתית — ולהפעיל אותו אוטומטית ב-boot.

תוצרים

  • שיעורי בית — חמש מעבדות:
    • lab-01-cli-tour — סיור מודרך ב-shell (ניווט, יצירה, צפייה, מחיקה).
    • lab-02-pipes-and-path — הרכבת פקודות עם pipes ו-globs; שינוי PATH.
    • lab-03-boot-and-fhs — בדיקת לוח הזמנים של ה-boot, רשימת שירותי systemd, סיור ב-FHS.
    • lab-04-permissions-and-users — יצירת משתמשים, הגדרת בעלות קבוצתית, התנסות עם SUID/SGID, תרחישי chmod.
    • lab-05-storage-and-services — mount של loopback device, עריכת /etc/fstab, כתיבת .service + .timer וצפייה בפעולתם.
  • פרויקט גמר — להפוך מכונת לינוקס לשלכם — להקים מכונת לינוקס חדשה, להתאים אותה מקצה לקצה, להוסיף משתמש ושירות systemd מותאם, למאונט volume, לבנות דוח מערכת עם pipes, ולהגיש את הכל כריפו Git (כ-3–5 שעות).

משאבים

  • מצגת — slides/linux-fundamentals/ — הריצו pnpm dev וגשו ל-/s/linux-fundamentals.
  • דף עזר — docs/cheat-sheet.html — דף הפניה אחד בעברית/אנגלית. פתחו בדפדפן, הדפיסו ל-PDF במידת הצורך.
  • קריאה חיצונית:
    • Linux Filesystem Hierarchy Standard (FHS) — ההפניה הקנונית.
    • The Linux Documentation Project — מדריכים ארוכים.
    • man hier — מתאר את מבנה מערכת הקבצים ישירות מהטרמינל שלכם.
    • The Linux Command Line (William Shotts, PDF חינמי) — ספר ליווי מצוין.

הערכה

סטודנט עובר את הקורס כאשר השלים:

  1. את כל חמש המעבדות (תוצרים מוגשים כתעתיק פקודות + רפלקציה קצרה).
  2. את פרויקט הגמר: סביבת לינוקס עובדת עם ההתאמות המתוארות, בתוספת סיכום קצר של רצף ה-boot ושל שירות ה-systemd שהוסיף.
  3. הוכיח, בשיחה, שהוא יכול לענות: "מה קורה כשאני לוחץ Enter על ls -l /etc?" — נוגע ב-shell, ב-PATH, ב-syscalls ובקרנל.
